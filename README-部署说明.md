# 多账号邮箱隔离版

## 已实现

- 每个登录账号只能看到自己创建的邮箱。
- 创建邮箱时，会在 `ADMIN_USERS` KV 的 `mailbox-meta:*` 记录中保存所属账号。
- 查看邮箱列表、读取邮件列表、读取邮件正文、续期、删除，全部进行服务端归属校验。
- 仅修改前端请求无法绕过权限校验。
- 删除子账号时，该账号名下邮箱自动转交给主账号。
- 原有专属查看链接仍按 token 工作，不受后台账号隔离影响。

## 旧数据迁移规则

升级前已经存在、但没有所属账号记录的邮箱，首次读取时会统一归属到主账号 `APP_USERNAME`。
这是因为旧版本没有保存创建者，无法可靠判断旧邮箱是由哪个子账号创建的。

## 部署

在本目录执行：

```bash
npm install
npx wrangler deploy
```

也可以只使用本目录中的 `worker.js` 替换原项目文件后重新部署。

## 必需配置

继续使用现有配置即可（密钥格式）：

- `ADMIN_USERS` KV 绑定
- `APP_USERNAME` 密码
- `APP_PASSWORD` 账号
- `MOEMAIL_API_KEY` 邮件密钥
- 建议配置 `PUBLIC_LINK_SECRET` 随意填写
